# Contributing to Kongvael Holdings

Thank you for your interest in contributing! We welcome all contributions.

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. **Install** dependencies with `npm install`
4. **Create a branch** for your feature: `git checkout -b feature/your-feature`
5. **Make your changes** and test locally with `npm run dev`
6. **Build** to verify: `npm run build`
7. **Commit** with clear, descriptive messages
8. **Push** to your fork and open a **Pull Request**

## Code Style

- Use consistent indentation (4 spaces)
- Follow existing naming conventions (BEM for CSS, PascalCase for components)
- Keep components focused and single-purpose
- Use CSS custom properties from `index.css` for colors, spacing, and typography
- No inline styles unless dynamic (e.g., colors from data)

## Reporting Issues

- Use [GitHub Issues](https://github.com/Saff9/KongvaelHoldings/issues)
- Include steps to reproduce, expected vs actual behavior
- Screenshots are helpful for UI issues

## Questions?

Open an issue or reach out via the contact form on the website.
